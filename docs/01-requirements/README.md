# Requirements

本目录把 Plasmod 的问题定义转换为可追踪的工程要求。需求编号不是未来承诺；只有在
[`requirements-traceability.md`](requirements-traceability.md) 中关联到实现和验证证据的条目，
才可视为已有实现。

## 阅读顺序

1. [`problem-statement.md`](problem-statement.md)：为什么需要 Agent-native 数据层。
2. [`stakeholders-and-use-cases.md`](stakeholders-and-use-cases.md)：谁在什么流程中使用系统。
3. [`functional-requirements.md`](functional-requirements.md)：系统必须提供的行为。
4. [`non-functional-requirements.md`](non-functional-requirements.md)：持久性、并发、可观测性等约束。
5. [`constraints-and-non-goals.md`](constraints-and-non-goals.md)：当前边界。
6. [`requirements-traceability.md`](requirements-traceability.md)：需求到代码、接口和测试的映射。
